## Small Example

If you want the Material style for input, copy:

- <a href="https://github.com/javadbat/design-system/blob/main/docs/styles/material.css">docs/styles/material.css</a>
- <a href="https://github.com/javadbat/design-system/blob/main/modules/jb-input/stories/styles/style-material.css">modules/jb-input/stories/styles/style-material.css</a>

Then load them in your app CSS.

```css
@import "./styles/material.css";
@import "./components/jb-input/style-material.css";

.settings-form {
  /* Scope the theme to one page, form, or demo wrapper. */
}
```

```html
<section class="settings-form material-style">
  <jb-input class="material-input" label="Project name"></jb-input>
</section>
```

## Composed Components

Some components reuse other components internally. Copy the child recipes too, so the full surface stays consistent.

For example, date input uses input, calendar, and popover styles:

```css
@import "./styles/material.css";
@import "./components/jb-input/style-material.css";
@import "./components/jb-calendar/style-material.css";
@import "./components/jb-popover/style-material.css";
@import "./components/jb-date-input/style-material.css";
```

## What To Customize

Start from shared tokens when the whole theme should change:

```css
.my-brand-theme {
  --jb-primary: #2563eb;
  --jb-primary-hover: #1d4ed8;
  --jb-surface: #ffffff;
  --jb-border-color: #dbe3ef;
  --jb-radius: 0.75rem;
}
```

Use component variables only when one component needs special treatment:

```css
.my-brand-theme jb-input {
  --border-color: var(--jb-border-color);
  --focus-border-color: var(--jb-focus-color);
}
```

Use <code>::part()</code> only for official parts exposed by that component. If a component does not expose a part, style it with variables first.
