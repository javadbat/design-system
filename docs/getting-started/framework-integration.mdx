# Framework Integration

JB Design System components are standard custom elements. Import the package once so the browser registers the element, then use the tag in your template.

For the full platform behavior, start with MDN's [Using custom elements](https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_custom_elements) guide.

Use this path for static pages, CMS templates, server-rendered pages, and small apps without a bundler.

```html
<script type="module" src="/assets/jb-input.js"></script>

<jb-input label="Full name" name="fullName"></jb-input>
```

If you use npm and a bundler, import the package instead:

```js
import "jb-input";
```

Import the web component package once, then allow dash-case custom elements with Angular's `CUSTOM_ELEMENTS_SCHEMA`.

```ts
// main.ts
import "jb-input";
```

```ts
import { Component, CUSTOM_ELEMENTS_SCHEMA } from "@angular/core";

@Component({
  selector: "app-profile-form",
  standalone: true,
  schemas: [CUSTOM_ELEMENTS_SCHEMA],
  template: `<jb-input label="Full name" name="fullName"></jb-input>`,
})
export class ProfileFormComponent {}
```

Read more in Angular's [`CUSTOM_ELEMENTS_SCHEMA`](https://angular.dev/api/core/CUSTOM_ELEMENTS_SCHEMA) reference.

Import the package once, then tell Vue's compiler which tags are custom elements.

```ts
// main.ts
import "jb-input";
```

```ts
// vite.config.ts
import vue from "@vitejs/plugin-vue";

export default {
  plugins: [
    vue({
      template: {
        compilerOptions: {
          isCustomElement: (tag) => tag.startsWith("jb-"),
        },
      },
    }),
  ],
};
```

```vue
<template>
  <jb-input label="Full name" name="fullName"></jb-input>
</template>
```

Read more in Vue's [Vue and Web Components](https://vuejs.org/guide/extras/web-components.html) guide.

Import the package once in your app entry or page component, then use the custom element tag directly in JSX.

```tsx
import "jb-input";

export function ProfileField() {
  return <jb-input label="Full name" name="fullName" />;
}
```

For TypeScript projects, add the tag to Solid's JSX namespace when you want typed custom-element attributes.

```ts
declare module "solid-js" {
  namespace JSX {
    interface IntrinsicElements {
      "jb-input": {
        label?: string;
        name?: string;
      };
    }
  }
}
```

Use a ref when you need to call DOM methods or assign complex properties.

```tsx
let field!: HTMLElement;

<jb-input ref={field} label="Full name" name="fullName" />;
```

Read more in Solid's [TypeScript](https://docs.solidjs.com/configuration/typescript) and [Refs](https://docs.solidjs.com/concepts/refs) docs.

In the App Router, register the web component from a Client Component because the package needs browser-side JavaScript to define the custom element.

```tsx
"use client";

import "jb-input";

export function ProfileField() {
  return <jb-input label="Full name" name="fullName" />;
}
```

Then render that Client Component from a page or Server Component.

```tsx
import { ProfileField } from "./profile-field";

export default function Page() {
  return <ProfileField />;
}
```

If you publish a prebuilt ESM file instead of importing from npm, load it with Next's `Script` component.

```tsx
import Script from "next/script";

export default function Page() {
  return (
    <>
      <Script src="/assets/jb-input.js" type="module" strategy="afterInteractive" />
      <jb-input label="Full name" name="fullName" />
    </>
  );
}
```

Read more in Next.js' [`use client`](https://nextjs.org/docs/app/api-reference/directives/use-client) and [`Script`](https://nextjs.org/docs/app/api-reference/components/script) docs.

Astro is a great fit for web components because an `.astro` page can ship mostly static HTML and add a small module script to register the custom element.

```astro
---
---

<script>
  import "jb-input";
</script>

<jb-input label="Full name" name="fullName"></jb-input>
```

Astro processes local scripts, bundles npm imports, and outputs module scripts automatically. Use this approach when the component package is installed from npm.

If you already have a compiled browser file, reference it from `public/` instead.

```astro
<script src="/assets/jb-input.js"></script>

<jb-input label="Full name" name="fullName"></jb-input>
```

Read more in Astro's [client-side scripts](https://docs.astro.build/en/guides/client-side-scripts/) docs.

For modern WordPress sites, load your compiled ESM file with `wp_enqueue_script_module`.

```php
add_action( 'wp_enqueue_scripts', function () {
    wp_enqueue_script_module(
        'jb-input',
        get_theme_file_uri( '/assets/jb-input.js' ),
        array(),
        '1.0.0',
        array( 'in_footer' => true )
    );
} );
```

Then use the element in a block, template, shortcode output, or theme file.

```html
<jb-input label="Full name" name="fullName"></jb-input>
```

Read more in WordPress' [`wp_enqueue_script_module()`](https://developer.wordpress.org/reference/functions/wp_enqueue_script_module/) and [`wp_enqueue_script()`](https://developer.wordpress.org/reference/functions/wp_enqueue_script/) references.

The same pattern applies almost everywhere:

1. Load or import the package once.
2. Use the `jb-*` tag in markup.
3. Pass simple data through attributes.
4. Use properties, refs, or DOM methods for complex values.
5. Listen to DOM events with the framework's native event binding.

For framework compatibility details, see [Custom Elements Everywhere](https://custom-elements-everywhere.com/).
