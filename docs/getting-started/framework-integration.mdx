### Load from a CDN

Use the UMD builds with regular `<script>` tags. Load the shared dependencies first, then load the component:

```html
<script src="https://unpkg.com/jb-validation@1.0.0/dist/jb-validation.umd.js"></script>
<script src="https://unpkg.com/jb-core@0.30.0/dist/index.umd.js"></script>
<script src="https://unpkg.com/jb-core@0.30.0/theme/dist/index.umd.js"></script>
<script src="https://unpkg.com/jb-core@0.30.0/i18n/dist/index.umd.js"></script>
<script src="https://unpkg.com/jb-input@3.17.0/web-component/dist/index.umd.js"></script>

<jb-input label="Full name" name="fullName"></jb-input>
```

### Host the files yourself

If you do not want to use a CDN, install the package and its dependencies:

```sh
npm install jb-input
```

Then copy these UMD files from `node_modules` into your public assets directory:

```text
node_modules/jb-validation/dist/jb-validation.umd.js
node_modules/jb-core/dist/index.umd.js
node_modules/jb-core/theme/dist/index.umd.js
node_modules/jb-core/i18n/dist/index.umd.js
node_modules/jb-input/web-component/dist/index.umd.js
```

The example below renames the files for clarity. Keep the same script order and update each `src` to match its copied location:

```html
<script src="/assets/jb-validation.umd.js"></script>
<script src="/assets/jb-core.umd.js"></script>
<script src="/assets/jb-core-theme.umd.js"></script>
<script src="/assets/jb-core-i18n.umd.js"></script>
<script src="/assets/jb-input.umd.js"></script>

<jb-input label="Full name" name="fullName"></jb-input>
```

If you use npm and a bundler, import the package instead:

```js
import "jb-input";
```

```tsx
import "jb-input";

export function ProfileField() {
```

For TypeScript projects, add the tag to Solid's JSX namespace when you want typed custom-element attributes.

```ts
declare module "solid-js" {
  namespace JSX {
    interface IntrinsicElements {
      "jb-input": {
        label?: string;
        name?: string;
      };
    }
  }
}
```

Use a ref when you need to call DOM methods or assign complex properties.

```tsx
let field!: HTMLElement;

<jb-input ref={field} label="Full name" name="fullName" />;
```

Read more in Solid's [TypeScript](https://docs.solidjs.com/configuration/typescript) and [Refs](https://docs.solidjs.com/concepts/refs) docs.

Import the package once to register the element, then use it in a Lit template. Lit's property and event bindings work directly with custom elements.

```ts
import { LitElement, html } from "lit";
import "jb-input";

export class ProfileField extends LitElement {
  render() {
    return html`
      <jb-input
        label="Full name"
        name="fullName"
        @input=${(event: InputEvent) => console.log(event.target)}
      ></jb-input>
    `;
  }
}

customElements.define("profile-field", ProfileField);
```

Use Lit's `.property=${value}` syntax when assigning objects, arrays, or other non-string properties. Read more in Lit's [expressions](https://lit.dev/docs/templates/expressions/) documentation.

Blazor can render JB custom elements directly in Razor components. Load the UMD scripts once in the app's host document, before the Blazor startup script. Use `wwwroot/index.html` for a Blazor WebAssembly app or `Components/App.razor` for a Blazor Web App.

```html
<script src="https://unpkg.com/jb-validation@1.0.0/dist/jb-validation.umd.js"></script>
<script src="https://unpkg.com/jb-core@0.30.0/dist/index.umd.js"></script>
<script src="https://unpkg.com/jb-core@0.30.0/theme/dist/index.umd.js"></script>
<script src="https://unpkg.com/jb-core@0.30.0/i18n/dist/index.umd.js"></script>
<script src="https://unpkg.com/jb-input@3.17.0/web-component/dist/index.umd.js"></script>
```

Keep the app's existing Blazor startup script after those component scripts. A Blazor WebAssembly app uses:

```html
<script src="_framework/blazor.webassembly.js"></script>
```

A Blazor Web App uses:

```html
<script src="_framework/blazor.web.js"></script>
```

After the scripts register the element, use it like regular HTML in any `.razor` file:

```razor
<jb-input label="Full name" name="fullName"></jb-input>
```

Pass simple values with HTML attributes. For complex JavaScript properties, DOM methods, or custom events, use Blazor's JavaScript interop from a collocated JavaScript module.

To self-host the scripts in Blazor, copy the same UMD files into a directory under `wwwroot` and reference them with local URLs.

Read more in Microsoft's [JavaScript location](https://learn.microsoft.com/aspnet/core/blazor/javascript-interoperability/location-of-javascript) and [JavaScript interoperability](https://learn.microsoft.com/aspnet/core/blazor/javascript-interoperability/) documentation.

```php
add_action( 'wp_enqueue_scripts', function () {

        get_theme_file_uri( '/assets/jb-input.js' ),
```

  Alpine.js And HTMX
</h2>

Both libraries enhance regular HTML, so load the JB UMD scripts first and use the custom element directly. Alpine can listen to its DOM events with `@event`, while HTMX can include it in ordinary forms.

```html
<div x-data="{ fullName: '' }">
  <jb-input
    label="Full name"
    name="fullName"
    @input="fullName = $event.target.value"
  ></jb-input>
</div>
```

```html
<form hx-post="/profile" hx-target="#result">
  <jb-input label="Full name" name="fullName"></jb-input>
  <button type="submit">Save</button>
</form>

<div id="result"></div>
```

Read more in Alpine's [`x-on`](https://alpinejs.dev/directives/on) and HTMX's [`hx-post`](https://htmx.org/attributes/hx-post/) documentation.

  Other Frameworks And CMSs
</h2>

The same pattern applies almost everywhere:

1. Load or import the package once.
2. Use the `jb-*` tag in markup.
3. Pass simple data through attributes.
4. Use properties, refs, or DOM methods for complex values.
5. Listen to DOM events with the framework's native event binding.

For framework compatibility details, see [Custom Elements Everywhere](https://custom-elements-everywhere.com/).
