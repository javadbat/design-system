# JB Design System ✨

JB Design System is a collection of focused UI packages for building multilingual, RTL-ready web applications. Components are written as web components so they work across JavaScript frameworks, and most packages include a React wrapper for React projects.

## What You Get 💡

- 🧩 Framework-friendly web components.
- ⚛️ React wrappers available from `package-name/react`.
- 📝 Form controls, pickers, validation, overlays, feedback, and high-interaction product UI.
- 🌐 First-class RTL, Persian locale, Persian calendar, and Persian number support.
- 🎨 Theme customization through CSS custom properties, CSS parts, and component states.
- 📦 Independent packages so teams can adopt only what they need.

## Start With One Package ⚡

Install the component closest to the user task.

```bash
npm install jb-input
```

Use it as a web component:

```js
import "jb-input";
```

```html
<jb-input label="First name" name="firstName"></jb-input>
```

Or use the React wrapper:

```jsx
import { JBInput } from "jb-input/react";

export function ProfileField() {
  return <JBInput label="First name" name="firstName" />;
}
```

## Build A Small Form 🧱

Most form components support labels, messages, validation, disabled states, styling hooks, and native form participation.

```jsx
import { JBForm } from "jb-form/react";
import { JBInput } from "jb-input/react";
import { JBMobileInput } from "jb-mobile-input/react";
import { JBButton } from "jb-button/react";

export function SignupForm() {
  return (
    <JBForm onSubmit={(event) => console.log(event.target.value)}>
      <JBInput name="fullName" label="Full name" required />
      <JBMobileInput name="mobile" label="Mobile number" required />
      <JBButton type="submit">Create account</JBButton>
    </JBForm>
  );
}
```

## Architecture 🏗️

JB Design System is not a single all-in-one package. Each component has its own package and repository. This lets product teams install only the components they need, update packages independently, and keep application bundles smaller.

Every component starts from the web platform:

- 🧩 The web component is the core implementation.
- ⚛️ The React wrapper maps React props, events, and refs onto the web component.
- 🧰 Shared utilities live in packages such as `jb-core`, `jb-core/i18n`, `jb-core/theme`, `jb-validation`, and `jb-form`.

## Styling 🎨

Because components use shadow DOM, customization is exposed through standard browser styling APIs.

- 🎨 Use the [color system](?path=/docs/theme-colors--docs) for palette tokens.
- 📐 Use the [sizing system](?path=/docs/theme-sizes--docs) for spacing and size tokens.
- 🧵 Use component CSS variables for local customization.
- 🧩 Use `::part(...)` and component states when a component exposes structured styling hooks.

```css
:root {
  --jb-primary: oklch(54% 0.18 250);
  --jb-input-border-radius: 8px;
}
```

## Internationalization 🌐

By default, components read locale from the `html` element `lang` attribute. Set the document language and direction at the application shell.

```html
<html lang="fa" dir="rtl"></html>
```

You can also control locale through `jb-core/i18n`.

```js
import { i18n } from "jb-core/i18n";

i18n.setLocale(new Intl.Locale("fa", {
  calendar: "persian",
  numberingSystem: "arabext",
}));
```

Use the Storybook toolbar to preview components in English, Persian, Persian numbers, LTR, and RTL.
