## Features
All form elements are implementing `WithValidation<TValidationValue>`, `JBFormInputStandards<TValue>` interface. means they have internal validation mechanism using `jb-validation` module and standard form elements mandatory features.

## Validation
All JB Design System form components has a readonly property `validation` in their dom.

```ts
// you can access validation module in any component like this.
document.querySelector('jb-input').validation;
```
> it's not just `jb-input` you can also get this property from all form elements.

By accessing this module you can add your own custom validation item or bind validation getter callback or even check and report validity of the form elements you are using.
for more information read [jb-validation](?path=/docs/components-jbvalidation-readme--docs) document.

## JBFormInputStandards
it's a standard interface that all jb design system modules must follow so they can take part in `jb-form`. if you you want your own custom module also works in `jb-form` you just have to implement this interface.
this interface make sure your component has `value`, `disable`, `required`, `isDirty`, `initialValue` ,... properties that needed by `jb-form` to takes you a part in group actions like checking dirty status of the form or get and set value and initial value of the form.
